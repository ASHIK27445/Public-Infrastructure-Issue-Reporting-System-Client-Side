# CommunityFix

A modern web-based platform for community issue reporting, management, and civic event engagement. CommunityFix enables citizens to report local issues, track their resolution progress, participate in community events, and engage with community improvements through an intuitive interface.

## Overview

CommunityFix is a comprehensive civic engagement platform that bridges the gap between citizens and local authorities. The platform allows users to report community issues such as infrastructure problems, public safety concerns, and environmental hazards. With role-based access control, the system supports three user types: Citizens, Staff, and Administrators, each with specific capabilities to ensure efficient issue resolution.

Beyond issue reporting, CommunityFix now features a full **Community Event & Volunteer Management System** — citizens can register as volunteers or guests, donate to events, participate for free without login, receive QR-based attendance certificates, and more.

## Community Event System (Steps 1–6)

CommunityFix includes a complete volunteer event management pipeline built across six steps:

### Step 1 — Event Creation
Admins create community events (Cleanup, Plantation, Repair, Awareness, Student Day, Meetup) via a 5-step form. Events can be linked to existing issues, support registration fees, guest limits, T-shirt collection, free participation, and fund goals.

### Step 2 — Volunteer Registration
Citizens register as volunteers or guests. The system handles:
- Capacity checks with automatic waitlisting
- Stripe payment sessions for paid events
- QR token generation and email confirmation
- Waitlist promotion with refund when a spot opens
- Free participants (no login required) via a separate flow

### Step 3 — Public Event Detail Page
Each event has a public page showing details, volunteer list, donation progress bar, reactions, discussion comments, and a donation widget.

### Step 4 — Admin Event Management
A dedicated admin panel per event shows confirmed volunteers, waitlist, donations, inline event editing, status management (with auto-refund on cancel), and a spending breakdown tab for completed events.

### Step 5 — QR Check-in Scanner
A mobile-optimized scanner page for event day attendance:
- Rear camera QR scan via `html5-qrcode`
- Manual email check-in fallback
- Supports all three participant types (volunteer, guest, free participant)
- Live attendance counter with auto-refresh every 15 seconds
- Recent check-ins and pending list tabs
- Admin-only undo check-in

### Step 6 — Digital Certificate Generation
After an event is marked completed:
- Admin clicks "Generate Certificates" — batch PDF generation runs in the background
- Puppeteer renders a beautiful A4 landscape certificate per attendee
- PDFs are uploaded to Cloudinary and emailed automatically via Nodemailer
- Each certificate has a unique ID (e.g. `CERT-2025-A3F9`) and a public verify URL
- Users view their certificates on `/my-certificates`
- Anyone can verify a certificate at `/verify/:certId`
- One-click LinkedIn "Add to Certifications" integration

## Installation

### Prerequisites
```bash
Node.js 18+
npm or yarn
MongoDB
Firebase Account
Stripe Account
Cloudinary Account
Google Gemini API Key
```

### Setup

1. **Clone Repository**
```bash
git clone https://github.com/yourusername/communityfix.git
cd communityfix
```

2. **Install Frontend Dependencies**
```bash
npm install
```

3. **Install Backend Dependencies**
```bash
cd server
npm install
```

4. **Frontend Environment Configuration**

Create `.env` file in the frontend root:
```env
VITE_API_URL=your_backend_api_url
VITE_FIREBASE_API_KEY=your_firebase_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_firebase_auth_domain
VITE_FIREBASE_PROJECT_ID=your_firebase_project_id
VITE_STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key
```

5. **Backend Environment Configuration**

Create `.env` file in the server root:
```env
PORT=5000
DB_USER=your_mongodb_username
DB_PASS=your_mongodb_password
FB_KEY=your_firebase_service_account_base64
FRONTEND_URL=http://localhost:5173
CLIENT_URL=http://localhost:5173
stripe_secretKey=your_stripe_secret_key
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your_gmail_address
EMAIL_PASS=your_gmail_app_password
EMAIL_FROM="CommunityFix <noreply@communityfix.com>"
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
GEMINI_API_KEY=your_gemini_api_key
```

6. **Run Development Server**
```bash
# Frontend
npm run dev

# Backend
cd server && node index.js
```

7. **Build Production**
```bash
npm run build
```
